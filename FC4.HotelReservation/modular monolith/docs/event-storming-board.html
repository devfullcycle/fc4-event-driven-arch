🟡 Agregados & Ciclos de Vida

🟡 Reservation (EventSourced)

Create → Pending → ┬→ Paid → Refunded
                              ├→ Cancelled
                              └→ Rejected

🟡 RoomTypeInventory (EventSourced)

Create → (Total=N, Reserved=0)

ReserveRooms(qty) → Reserved += qty
ReleaseRooms(qty) → Reserved -= qty

🟡 Payment (AggregateRoot)

Create → Pending → Processing → ┬→ Completed → Refunded
                                                       └→ Failed

🟡 GuestInfo (Read Replica)

Projeção local do módulo Guests
Campos: Email, FullName
🔗 Comunicação entre Módulos (Integration Events via MassTransit)
Evento de Integração Produtor Consumidores
ReservationCreated Reservations Payments (cria pagamento) · Reservations/Consumers (projeta no MongoDB)
ReservationStatusChanged Reservations Reservations/Consumers (atualiza status no MongoDB)
InventoryChanged Reservations Reservations/Consumers (atualiza inventário no MongoDB)
PaymentStatusChanged Payments Reservations (atualiza status: Paid / Rejected / Cancelled)
